## 修改 ①：原始小波基函数生成代码的问题与修复

**位置**：`src/daubechies.cpp` 的 `cascade_scaling_function`

**原始代码的问题**：
原版用"从 Kronecker δ 出发、迭代 `level + 12` 次的级联算法"计算尺度函数 φ(x)。
该算法**每迭代一次数组长度翻倍**，递推关系为 `nc(下一次) = 2·nc(本次) + (2N-1)`。
在 `level = 14` 时迭代 26 次，最终数组规模：

| 参数 | 最终数组元素数 | 计算峰值内存 |
|------|---------------|-------------|
| N=6  | ≈ 8 亿        | ≈ 16 GB     |
| N=10 | ≈ 13 亿       | ≈ 27 GB     |

这导致在内存不足的机器上**直接崩溃（堆损坏 0xc0000374）**，在内存充足的机器上也极慢。

**修复方案**：改为**固定分辨率的级联迭代**。网格大小始终固定为目标分辨率 n_out，
迭代时只在两个等大数组间倒换：

```
φ_new[i] = √2 · Σ_k h_k · φ_old[ 2i - k·2^level ]
```

- 内存从 ~16 GB 降到 ~3 MB（约 5000 倍），与机器内存无关；
- 仍是迭代格式，保留级联的收敛/自纠错特性；
- 以整数点特征值为初值（整数节点是迭代的精确不动点），收敛极快。

## 修改 ②：连接系数改为精确解（LRT 方法）

**位置**：
- `src/daubechies.cpp` 新增 `connection_coeff_d1d1`
- `src/wavelet_fem.cpp` 的 `assemble_1d_matrices`

**原始做法的问题**：
原版用级联点值 + Simpson 数值积分计算质量/刚度连接系数
`∫ φ^(n)(y-k) φ(y) dy`。由于 Daubechies φ 的分形/低光滑性，标准数值积分公式
对其收敛极慢且不稳定；尤其 φ'' 还用四阶差分逐点求，对低阶 N 更不可靠。

**修复方案**：采用 **Latto–Resnikoff–Tenenbaum (LRT)** 方法，把连接系数转化为代数问题，
完全不依赖 φ 的逐点采样与数值积分。

二阶连接系数 Γ_d = ∫ φ'(x)φ'(x-d) dx 满足由双尺度方程导出的特征方程：

```
Γ_d = 4 · Σ_{m,n} h_m h_n · Γ_{2d + n - m}      （即 Γ = TΓ，特征值 1）
```

再用矩条件 `Σ_d d² Γ_d = -2` 唯一定标。只需用 Eigen 解一个 (4N-3)×(4N-3) 的小型稠密特征问题。

应用于矩阵组装：
- **内部基函数**（支撑完全在区间内）：
  - 质量矩阵由 Daubechies 正交归一性精确给出：`∫φ(x-k)φ(x-l)dx = δ_{kl}`；
  - 刚度矩阵由 LRT 精确给出：`∫φ''(y-k)φ(y-l)dy = -Γ_{k-l}`（分部积分）。
- **边界基函数**（支撑被区间截断）：暂仍用原数值积分（截断连接系数），
  这是后续可继续改进的部分（Behiry 2001 有界区间法）。

> 附注：`src/wavelet_fem.cpp` 中的 `apply_boundary_conditions` 也作了修正
> （边界方程原本只用单个基函数、且漏掉导数的 2^j 缩放，导致系统接近奇异、解发散）。
> 此修正是上述"修改后输出数值"能保持有界、不再发散（原版第一步即爆炸到 10⁷⁰）的前提，故一并包含。

## 修改 ③：修改后的输出数值

`results/` 中为应用上述修改后、运行算例 1（Robin 边界，N=6, j=4, T=1.0）产生的结果。

**数值特征对比**：

| | 原始版本（git 提交的结果） | 本次修改后 |
|---|---|---|
| t=0（初始投影） | 正常，O(0.1) | 正常，O(0.1) |
| 第一个时间步 | **爆炸到 10⁷⁰** | 有界，O(0.1) |
| t=1.0 | 10¹⁵³（发散） | 有界，O(0.01–0.1) |

即：原版从第一个时间步起就发散（这也是原 README §8.1 记录的"已知未解决问题"），
修改后解全程保持**有界**、可正常可视化。

**仍存在的局限**：误差暂未随网格加密（j 增大）而下降，根源在边界处理
（截断连接系数 + 边界条件施加），属于后续工作。
